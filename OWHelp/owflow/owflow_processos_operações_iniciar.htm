Iniciar Processo

Esta secção contém apenas informação acerca dos procedimentos para iniciar um Processo no OWFlow, e de todos os procedimentos relacionados com isso. Não contém informação acerca de mais nenhum módulo.  
 

Um processo pode ser iniciado a partir de uma Distribuição do Registo ou directamente a partir do OWFlow.

Existem dois tipos diferentes de processos:

    1. Processos Ad-hoc não baseados em fluxos predefinidos
    2. Processos baseados em fluxos predefinidos

 

A) Fluxo Novo

Permite iniciar um processo Ad-hoc sem fluxo predefinido.

A diferença com o Iniciar processo a partir de Fluxo Existente do iniciar processo através de Fluxo Ad-hoc é que, no fluxo Ad-hoc, não basta despachar a primeira etapa mas tem que se definir as etapas, prazos, alarmes e destinatários. No Fluxo Existente, pré definido, tudo isto já está definido e o processo passa automaticamente para a etapa seguinte. 

No processo Ad-hoc existe um campo para seleccionar a sequência de utilizadores (executantes), o campo de despacho e a possibilidade de anexar à primeira etapa ou ao processo os documentos necessários.

Para adicionar um executante (pode adicionar quantos forem necessários), basta seleccionar o botão de pesquisa  . Aparece o écran semelhante ao da figura que se segue. Seleccione os executantes (pode inserir de imediato uma lista de executantes) e confirme.

 

No processo Ad-hoc, o nome das etapas assume o nome do processo.

No separador Etapas é possível adicionar mais etapas ou editar as etapas existentes.

 

O utilizador pode especificar o Nome, o Prazo e o Executante de cada Etapa. Pode ainda adicionar Alarmes às etapas.  O código é sempre fixo e igual a ”ADHOC” e o Número e Ano são gerados automaticamente.

 

B) Fluxo Existente

Permite iniciar um processo a partir de um fluxo predefinido.  O utilizador deve primeiro seleccionar um fluxo da lista de  fluxos existentes em produção.  Esta lista apresenta todos os fluxos em produção em que o utilizador é executante da primeira etapa.

 

 Os campos código, número e ano são preenchidos pelo sistema e não podem ser alterados.  O código é copiado do fluxo, o ano é o corrente e é obtido da data do sistema do servidor e o campo número é sequencial. O processo é identificado no sistema pelo conjunto CÓDIGO -NÚMERO / ANO, por exemplo: VIAGEM-000014/1999. O número do processo só é mostrado depois de criado o processo.

Os campos nome do processo e prazo/data limite são preenchidos pelo sistema com base no nome do fluxo e no prazo limite. O utilizador pode alterar o nome do processo. O nome do processo é de preenchimento obrigatório.

Depois de confirmar é criado o processo com as etapas do fluxo e mostrado o número de processo atribuído.  O processo fica com o estado activo e surge a janela para “Despachar Processo” com a etapa inicial. Nesta etapa o utilizador que iniciou o processo pode anexar os documentos que deram entrada, e despachar para a etapa seguinte.
